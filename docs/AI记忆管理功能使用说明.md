# AI 交互与记忆管理功能使用说明

## 功能概述

本系统提供了两种 AI 对话记忆管理功能：

1. **手动压缩记忆** - 清理冗余对话，腾出上下文空间
2. **提取对话摘要** - 归档问诊记录，生成电子病历

---

## 功能详解

### [5] 手动压缩记忆

**作用**：这是一个"清理"动作。将冗长的历史对话浓缩成简短的总结，目的是**腾出上下文空间**，防止模型因内容过长而遗忘或报错。

**使用建议**：建议在对话 10-15 轮后执行，以防内容过长影响模型表现。

**执行后提示**：
```
[OK] 已为您清理了冗余的对话细节，保留了关键信息。这有助于 AI 在长对话中保持清醒。
```

---

### [6] 提取 AI 管家对话摘要

**作用**：这是一个"归档"动作。从当前的问诊记录中提取 5W 关键信息（Who, What, When, Where, Why），并保存到本地日志文件中，目的是**生成电子病历**。

**核心特性**：
- ✅ **即时生效**：无论对话长短，点击后立即尝试提取
- ✅ **兜底保障**：即使 LLM 判定信息不足，也会保存原始对话作为"简短问诊记录"
- ✅ **统一反馈**：永远显示正向的"已成功保存"提示

**执行后提示**：
```
[OK] 已成功提取本次问诊精华，并保存至 D:\chat-log 目录。您可以随时查阅宠物的健康档案。
```

如果信息不足，会显示：
```
[OK] 已成功保存本次问诊记录至 D:\chat-log 目录（简短问诊记录）。您可以随时查阅宠物的健康档案。
```

## 常见问题

### Q1: 为什么我点击 [6] 后显示的是"简短问诊记录"而不是 5W 信息？

**答**：这说明当前对话内容较少，LLM 无法提取足够的 5W 信息。但不用担心，系统已经为您保存了完整的原始对话记录，您可以在 `D:\chat-log\log.txt` 中查看。

### Q2: 我可以随时查看之前保存的病历吗？

**答**：可以！所有保存的记录都在 `D:\chat-log\log.txt` 文件中，按时间顺序排列，您可以随时用文本编辑器打开查看。

### Q3: 什么时候应该使用 [5] 手动压缩记忆？

**答**：当您与 AI 进行了长时间对话（超过 10-15 轮），感觉 AI 开始"忘记"之前的内容时，可以使用此功能清理历史对话，让 AI 保持"清醒"。

### Q4: 压缩记忆会删除我的病历记录吗？

**答**：不会！压缩记忆只会清理 AI 的短期上下文，不会影响已保存到 `D:\chat-log\log.txt` 的病历记录。您的所有问诊记录都会永久保存。
