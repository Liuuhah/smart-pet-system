# 期末答辩自查清单与要点总结

## 📋 一、核心数据结构应用（得分点）

在答辩中，老师最关注的是你是否真正理解并正确使用了数据结构。请确保你能清晰阐述以下几点：

### 1. 哈希表 (Hash Table) - `PetProfileManager`
*   **应用场景**：用于存储和快速检索宠物档案（`PetProfile`）。
*   **底层实现**：利用 Python 内置的 `dict`（字典），其底层采用开放寻址法解决冲突。
*   **复杂度优势**：实现了 O(1) 的增删改查效率。相比于数组或链表的 O(n) 查找，在处理大量宠物数据时性能优势显著。
*   **封装体现**：通过 `add_pet`, `get_pet` 等方法封装了底层操作，体现了 ADT（抽象数据类型）的设计思想。

### 2. 双向链表 (Doubly Linked List) - `HealthTimeline`
*   **应用场景**：管理每只宠物的健康记录时间线。
*   **设计理由**：
    *   **双向遍历**：支持从尾部向前遍历（`traverse_backward`），高效获取“最近 N 条记录”，这是单向链表无法做到的。
    *   **动态扩展**：健康记录数量不固定，链表比数组更节省空间且插入效率高（O(1)）。
*   **指针操作**：在代码中手动维护了 `prev` 和 `next` 指针，展示了扎实的指针操作功底。

## 💡 三、常见答辩问题预演

### Q1: 为什么选择哈希表而不是二叉搜索树？
**回答思路**：哈希表在平均情况下能达到 O(1) 的查找速度，而 BST 是 O(log n)。对于宠物 ID 这种唯一标识符的精确查找，哈希表更直接高效。

### Q2: 双向链表相比单向链表有什么额外开销？
**回答思路**：每个节点多了一个 `prev` 指针，增加了少量的内存开销。但换来的是反向遍历的能力，这在查看“最近历史记录”的场景下非常有价值。

### Q3: 你的 AI 顾问是如何与数据结构结合的？
**回答思路**：我采用了“数据驱动”的模式。先从哈希表中提取宠物基本信息，再从双向链表中抽取最近的健康记录，组装成上下文发送给 LLM。这样 AI 就能基于真实的历史数据给出个性化建议。
