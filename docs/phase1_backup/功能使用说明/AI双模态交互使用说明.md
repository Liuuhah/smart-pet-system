# AI 管家双模态交互使用说明

## 功能概述

AI 宠物健康管家现在具备**"双重人格"**，能够根据用户输入自动切换两种模式：

1. **闲聊模式** - 随和、亲切、像老朋友一样自然
2. **专业问诊模式** - 专业、严谨、提供结构化的医疗参考

---

## 使用指南

### 自动切换，无需手动操作

用户只需正常输入文本，系统会自动识别意图并选择合适的模式：

#### 闲聊模式示例

```
[你]: 你好呀

[AI 管家]: 你好呀！👋 这里是豆豆的专属宠物健康助手～今天想聊点什么？是豆豆又长胖了呢还是又有什么小状况啦？😄
```

**特点**：
- ✅ 简短、亲切
- ✅ 带表情符号
- ✅ 无医疗格式
- ✅ Temperature = 0.8（更灵动）

#### 专业问诊模式示例

```
[你]: 豆豆最近有点拉肚子

[AI 管家]: 听到豆豆拉肚子，主人一定很着急吧？别担心，我们一步步来分析。🐾

**初步诊断建议：**
3岁的博美犬出现腹泻，常见原因包括：
1. **饮食不当**（如吃多了、吃了不合适的食物）
2. **病毒感染**（细小病毒、冠状病毒等）
...

**⚠️ 紧急程度评估：**
- 如果豆豆精神尚可、食欲正常、无呕吐/便血，可先观察12-24小时
- 如果出现以下任一情况，请**立即就医**：
  ...

**家庭护理建议：**
1. **饮食调整**：暂停喂食3-6小时...
2. **补充水分**：腹泻容易脱水...
...
```

**特点**：
- ✅ 结构化内容
- ✅ 专业诊断建议
- ✅ 紧急程度评估
- ✅ 家庭护理指导
- ✅ Temperature = 0.4（更严谨）

## 触发条件

### 闲聊模式触发词（示例）
- "你好"、"在吗"、"早上好"
- "今天天气不错"
- "豆豆今天看起来很开心"
- 任何不包含医疗关键词的对话

### 专业问诊模式触发词
系统会检测以下医疗关键词（20+ 个）：

```
病、痛、吐、拉、吃、喝、药、医、诊、症状、
发烧、咳嗽、瘦、胖、呕吐、腹泻、精神、异常、
不吃、不喝、拉肚子、感冒、受伤、流血、抽搐
```

只要用户输入中包含以上任意关键词，就会自动进入专业问诊模式。

## 常见问题

### Q1: 为什么我输入"它看起来没精神"会进入问诊模式？

**A**: 因为"精神"在医疗关键词列表中。这是有意设计的，因为"没精神"可能是健康问题的征兆。

### Q2: 如何让 AI 在问诊时也保持轻松的语气？

**A**: 目前专业模式的语气已经设计为"专业且温和"。如果您希望更轻松的语气，可以在后续版本中调整 `context_instruction` 的内容。

### Q3: 我可以手动指定模式吗？

**A**: 当前版本不支持手动指定，系统会自动识别。未来可以考虑添加 `/chat` 或 `/consult` 指令来强制切换模式。

### Q4: Temperature 是什么？为什么要调整它？

**A**: Temperature 是控制 LLM 生成文本随机性的参数：
- **低温度（0.2-0.4）**：回复更确定、严谨，适合专业场景
- **高温度（0.7-0.9）**：回复更灵动、创造性，适合闲聊场景

## 技术说明

### 意图识别逻辑

系统使用基于关键词的轻量级意图识别：

```python
def _detect_intent(self, user_input: str) -> str:
    medical_keywords = ["病", "痛", "吐", "拉", ...]
    if any(kw in user_input for kw in medical_keywords):
        return "consult"  # 专业问诊模式
    return "chat"  # 闲聊模式
```

**优点**：
- ✅ 实现成本低
- ✅ 响应速度快（无需额外 LLM 调用）
- ✅ 效果显著

**局限**：
- ⚠️ 可能会漏掉一些隐晦的病情描述
- ⚠️ 某些非医疗场景可能误判（如"我今天吃了什么"）

### 动态 Prompt 注入

每次调用 `consult()` 时，系统会：

1. 识别意图
2. 根据意图选择合适的 `context_instruction`
3. 追加到 System Prompt 后面
4. 调用 LLM 生成回复
5. 恢复原始 System Prompt

这确保了每次对话都是独立的，不会相互影响。

### Temperature 调优

| 模式 | Temperature | 效果 |
|------|-------------|------|
| 闲聊 | 0.8 | 更灵动、创造性高、带幽默感 |
| 问诊 | 0.4 | 更严谨、确定性高、结构化强 |

## 开发者指南

### 扩展医疗关键词

如果发现某些医疗描述未被识别，可以在 `_detect_intent()` 方法中添加新词：

```python
medical_keywords = [
    "病", "痛", "吐", "拉", ...,
    "新词汇"  # 在这里添加
]
```

### 调整 Temperature

如果需要调整两种模式的 Temperature，可以修改 `consult()` 方法中的值：

```python
if intent == "chat":
    temperature = 0.8  # 调整这个值
else:
    temperature = 0.4  # 调整这个值
```

**建议范围**：0.2 - 0.9

### 自定义 System Prompt 后缀

如果需要调整 AI 的语气和行为准则，可以修改 `context_instruction` 的内容：

```python
if intent == "chat":
    context_instruction = "\n\n【当前状态：闲聊模式】\n..."  # 修改这里
else:
    context_instruction = "\n\n【当前状态：专业问诊模式】\n..."  # 修改这里
```
