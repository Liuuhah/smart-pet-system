# AI 咨询子模式与精准存档功能使用说明

## 功能概述

本次更新将主菜单 `[4] 咨询 AI 管家` 从单次问答升级为**连续对话子模式**,并新增了以下功能:

1. **多轮对话** - 支持连续提问,保持上下文记忆
2. **历史记录查看** - 使用 `/list` 指令查看所有对话轮次
3. **精准摘要提取** - 使用 `/archive` 指令提取指定轮次的问诊记录
4. **退出自动保存** - 输入 `exit` 时自动保存最后一段对话

---

## 使用指南

### 1. 进入连续对话模式

```
主菜单选择 [4] → 输入宠物姓名 → 开始对话
```

进入后会显示欢迎界面:

```
============================================================
   AI 宠物健康管家 - 连续对话模式
============================================================
   输入 'exit' 或 'quit' 退出
   输入 '/list' 查看历史记录
   输入 '/archive <轮次>' 精准提取摘要(如 /archive 1,3-5)
============================================================
```

### 2. 正常对话

直接输入您的问题即可:

```
[你]: 我的狗狗最近食欲不振

🤖 AI 管家思考中...

[AI 管家]: 建议观察一下是否有其他症状,如呕吐、腹泻等...
```

### 3. 查看历史记录 (/list)

输入 `/list` 可以查看所有对话轮次:

```
[你]: /list

--- Round 1 ---
[你]: 我的狗狗最近食欲不振
[AI]: 建议观察一下是否有其他症状...

--- Round 2 ---
[你]: 它还有点拉肚子
[AI]: 可能是肠胃问题,建议就医检查...

--- Round 3 ---
[你]: 好的,谢谢
[AI]: 不客气,如有恶化请及时就医
```

**注意**: 
- 每条消息超过 100 字符会自动截断显示
- 只显示完整的对话轮次(user + assistant)

### 4. 精准提取摘要 (/archive)

使用 `/archive` 指令可以提取指定轮次的问诊记录并保存到日志文件。

#### 支持的表达式格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 离散选择 | `/archive 1,3,5` | 提取第 1、3、5 轮 |
| 范围选择 | `/archive 1-3` | 提取第 1、2、3 轮 |
| 混合选择 | `/archive 1,3-5,8` | 提取第 1、3、4、5、8 轮 |

#### 示例

```
[你]: /archive 1,3

[精准提取] 正在提取第 [1, 3] 轮对话...
[系统] 已成功提取第 1, 3 轮对话精华,并保存至 D:\chat-log。
```

#### 错误处理

- **无效表达式**:
  ```
  [你]: /archive abc
  [系统] 无法识别您的指令,请使用格式:/archive 1,3-5
  ```

- **超出范围**:
  ```
  [你]: /archive 1,10
  [系统] 已成功提取第 1 轮对话精华,并保存至 D:\chat-log。
  注意:第 10 轮超出当前对话范围(共 5 轮),已忽略
  ```

### 5. 退出并自动保存

输入 `exit`、`quit`、`返回` 或 `q` 退出对话模式:

```
[你]: exit

[系统] 已结束本次咨询,正在为您保存记录...

--- 问诊记录 ---
【记录时间】2026-04-25 18:30:00
【对话轮次】第 3 轮
【提取模式】标准模式
...
----------------
[OK] 记录已保存,返回主菜单。
```

**注意**: 
- 即使只进行了一轮对话,退出时也会强制保存
- 如果 LLM 判定信息不足,会保存为"简短问诊记录"
- 按 `Ctrl+C` 中断时也会自动保存

## 常见问题

### Q1: 为什么我输入 `/archive` 后没有反应?

**A**: 请确保您指定了轮次表达式,正确格式为:
```
[你]: /archive 1,3-5
```

### Q2: 如何知道当前有多少轮对话?

**A**: 使用 `/list` 指令查看所有历史记录,最后会显示总轮数。

### Q3: 提取的摘要保存在哪里?

**A**: 所有提取结果均保存至 `D:\chat-log\log.txt`,您可以随时查阅。

### Q4: 如果我输入的轮次超出范围怎么办?

**A**: 系统会提取有效范围内的轮次,并给出警告提示,不会导致程序崩溃。

### Q5: 连续对话模式下,AI 会记住之前的对话吗?

**A**: 是的,`PetHealthAdvisor` 内部维护了 `chat_history` 列表,可以实现上下文记忆。

## 技术说明

### 架构设计

1. **模块化设计**:
   - 轮次解析、上下文构建、LLM 调用分离为独立方法
   - 便于单元测试和后续维护

2. **容错处理**:
   - 轮次表达式解析失败时返回空列表
   - 超出范围的轮次生成警告但不影响其他轮次
   - 自动交换 `start > end` 的范围表达式

3. **代码复用**:
   - 精准提取复用了现有的 `_extract_5w_info()` 和 `_save_to_log_file()` 方法
   - 避免重复代码,保持架构一致性

### 限制说明

- **Phase 1 限制**: 目前仅支持对内存中现存的对话进行提取,不支持从历史日志文件中回溯已压缩的内容
- **LLM 依赖**: 精准提取需要本地 LLM 服务正常运行(LM Studio + Qwen)
