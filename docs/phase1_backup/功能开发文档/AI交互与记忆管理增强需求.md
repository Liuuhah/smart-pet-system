# 智能宠物系统 - AI 交互与记忆管理功能增强需求

**版本**: Phase 1.1  
**状态**: ✅ 已完成  
**完成日期**: 2026-04-25  
**目标**: 优化 AI 管家模块的用户交互体验，明确“记忆压缩”与“对话摘要”的功能边界，并实现按需提取逻辑。

---

## 1. 核心需求说明

### 1.1 概念澄清与用户引导
由于“记忆压缩”和“对话摘要”属于较为底层的 LLM 管理功能，普通用户容易混淆。需要在用户触发相关操作时，提供清晰的解释性提示。

*   **[5] 手动压缩记忆**：
    *   **概念**：这是一个“清理”动作。将冗长的历史对话浓缩成简短的总结，目的是**腾出上下文空间**，防止模型因内容过长而遗忘或报错。
    *   **提示语要求**：在执行后，需向用户展示：“已为您清理了冗余的对话细节，保留了关键信息。这有助于 AI 在长对话中保持清醒。”
*   **[6] 提取 AI 管家对话摘要**：
    *   **概念**：这是一个“归档”动作。从当前的问诊记录中提取 5W 关键信息（Who, What, When, Where, Why），并保存到本地日志文件中，目的是**生成电子病历**。
    *   **提示语要求**：在执行后，需向用户展示：“已成功提取本次问诊精华，并保存至 `logs/medical_records/` 目录。您可以随时查阅宠物的健康档案。”

### 1.2 解除提取限制（按需提取）
*   **现状问题**：目前的底层逻辑设置了计数器（如每 5 条消息才尝试提取一次），导致用户在只进行了一轮重要问诊后无法立即保存摘要。
*   **改进要求**：当用户通过主菜单手动点击 `[6]` 时，应**强制跳过计数器限制**，立即调用 LLM 对当前所有对话进行 5W 信息提取并归档。

### 1.3 增加操作频率引导
*   **背景**：在 Phase 1 使用本地模型（如 LM Studio）时，上下文窗口有限。
*   **改进要求**：在主菜单的选项 `[5]` 后面增加注释提示：
    > `[5] 手动压缩记忆 (建议在对话 10-15 轮后执行，以防内容过长影响模型表现)`

## 2. 详细开发任务清单

### 任务一：增强交互反馈 (`main.py`)
1.  **修改菜单文案**：更新选项 `[5]` 的显示文本，加入频率建议。
2.  **优化执行反馈**：
    *   在 `compress_memory()` 成功后，打印关于“清理冗余、保留关键”的解释。
    *   在 `get_medical_summary()` 成功后，打印关于“归档病历、保存路径”的解释。

### 任务二：重构底层提取逻辑 (`chat_compress_client.py`)
1.  **暴露强制提取接口**：新增一个方法（如 `extract_summary_now()`），该方法内部直接调用 `_extract_5w_info` 但不检查 `_extraction_counter`。
2.  **优化失败处理（优先级提升）**：
    *   **问题现状**：目前手动触发提取时，如果 LLM 判定信息量不足会返回“失败”，导致用户看到“提取失败”的提示，体验极差。
    *   **改进要求**：**手动触发的优先级应设为最高**。只要用户点击了 `[6]`，无论对话长短或内容多少，都应视为一次成功的“归档尝试”。
    *   **实现逻辑**：在手动模式下，即使 LLM 返回“无法提取”，也应将当前原始对话记录保存至日志，并标记为“简短问诊记录”，而不是报错。确保用户看到的反馈永远是“已成功保存”。

### 任务三：同步更新业务层 (`pet_health_advisor.py`)
1.  **适配新接口**：确保 Advisor 类能正确调用 Client 的新增强制提取方法。
2.  **人设一致性**：确保这些系统级的提示信息不会破坏“专业宠物医生”的沉浸感（建议以系统助手的口吻弹出）。

## 3. 验收标准 (Definition of Done)

1.  **概念清晰**：用户按下 `[5]` 和 `[6]` 后，能通过控制台文字明确区分“清理内存”和“保存病历”的区别。
2.  **即时生效**：用户在第一轮问诊结束后即可通过 `[6]` 成功提取并看到日志文件生成。
3.  **引导到位**：主菜单清晰标注了记忆压缩的建议执行时机。
4.  **日志完整**：生成的 `logs/medical_records/` 文件包含完整的 5W 信息，且文件名具有时间戳标识。

### 5.2 核心技术亮点

1. **手动触发优先级最高**
   - 当用户通过菜单 `[6]` 手动触发时，完全绕过底层的“信息量不足”判定
   - 无论对话长短、内容多少，都会尝试提取并归档

2. **兜底归档策略**
   - 如果 LLM 返回“无法提取”，系统自动将原始对话保存为“📝 简短问诊记录（兜底归档）”
   - 日志文件包含完整的对话内容和说明文字，方便后续查阅

3. **统一反馈机制**
   - 所有路径下用户看到的都是正向反馈：“已成功保存..."
   - 消除了“提取失败”等负面提示，提升用户体验
