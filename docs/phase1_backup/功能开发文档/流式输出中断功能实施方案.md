# 流式输出中断（一键静音）功能实施方案

**版本**: Phase 1.7  
**状态**: ✅ 已完成  
**完成日期**: 2026-04-26  
**优先级**: 🔴 P0 (高优先级 - 提升交互体验)  
**目标**: 实现 AI 在流式输出过程中的即时中断功能，允许用户在不退出对话的情况下暂停冗长的回复。

---

## 1. 需求背景

### 1.1 痛点描述
*   **信息过载**：在专业问诊模式下，AI 有时会给出过于详尽的长篇大论，用户在阅读过程中感到压力。
*   **缺乏控制感**：目前的流式输出虽然能实时显示，但一旦开始就无法中途停止，用户必须等待全部内容输出完毕才能进行下一步操作。
*   **效率低下**：如果用户已经得到了想要的答案，却不得不等待 AI 说完剩下的“废话”，严重影响了快速迭代的测试效率。

### 1.2 核心目标
*   **即时响应**：按下指定按键（如 `Esc`）后，AI 必须在毫秒级时间内停止输出。
*   **上下文保留**：中断后，已经输出的内容必须保留在屏幕上，且完整存入 `chat_history`，确保后续对话逻辑连贯。
*   **零干扰**：监听机制不能占用 CPU 资源，不能影响 AI 正常的输出速度和流畅度。

---

## 2. 技术选型与架构设计

### 2.1 核心技术：多线程 + 非阻塞监听
采用 **Producer-Consumer（生产者-消费者）** 模型的变体：
*   **主线程（生产者）**：负责从 LM Studio 接收数据流并打印到终端。
*   **守护线程（消费者/监听者）**：负责在后台静默监听键盘输入。

### 2.2 关键库选择：`msvcrt` (Windows Built-in)
*   **选择理由**：
    *   **零依赖**：Python 内置，无需 `pip install`，确保环境纯净。
    *   **高性能**：直接调用 Windows API，开销极低，不会拖慢主线程。
    *   **权限友好**：不需要管理员权限即可在当前终端窗口内工作。

### 2.3 逻辑流程
1.  **初始化**：在发送请求前，设置全局标志位 `self.is_interrupted = False`。
2.  **启动监听**：开启一个 `daemon=True` 的后台线程，循环调用 `msvcrt.kbhit()`。
3.  **流式输出**：主线程在遍历 `response.iter_lines()` 时，每处理一个字符块都检查 `self.is_interrupted`。
4.  **触发中断**：若检测到按键，立即 `break` 跳出循环，打印 `[已暂停输出]`，并返回当前已生成的文本。

### 3.1 `src/ai_assistant/chat_compress_client.py` 修改方案

#### 任务 A：引入依赖
在文件头部导入必要的模块：
```python
import threading

```

#### 任务 B：新增 `_keyboard_listener` 方法
这是一个私有方法，作为后台线程的运行函数。
*   **逻辑**：无限循环检测 `msvcrt.kbhit()`。如果检测到按键且为 `Esc` (ASCII 27)，则将 `self.is_interrupted` 设为 `True` 并退出线程。

#### 任务 C：改造 `_send_chat_request` 方法
这是核心的流式输出逻辑，需要进行以下增强：
1.  **重置状态**：在请求开始前执行 `self.is_interrupted = False`。
2.  **启动线程**：创建并启动监听线程 `t = threading.Thread(target=self._keyboard_listener, daemon=True)`。
3.  **增加安检点**：在 `for chunk in response.iter_lines():` 循环内部增加判断：
    ```python
    if self.is_interrupted:
        print("\n[已暂停输出]", end="", flush=True)
        break
    ```
4.  **异常捕获**：确保即使发生中断，也能正确关闭网络连接并返回已生成的文本。

### 4.2 按键映射
*   **暂停键**：`Esc` (Escape)。
*   **理由**：该键位于键盘左上角，易于触达，且在常规对话中极少被用到，不会产生误触。

## 5. 验收标准 (Acceptance Criteria)

1.  **中断有效性**：
    *   在 AI 正在输出长文时按下 `Esc`，输出必须在 0.5 秒内完全停止。
2.  **内容完整性**：
    *   中断后，屏幕上保留已输出的所有文字。
    *   输入下一个问题时，AI 能根据之前的上下文（包括中断前的那部分）正常回答。
3.  **性能无损耗**：
    *   在未按下暂停键的情况下，AI 的输出速度与优化前保持一致，无明显卡顿。
4.  **稳定性**：
    *   连续多次触发暂停和继续对话，程序不崩溃、不卡死。

---

## 6. 风险提示与对策

*   **风险**：`msvcrt` 仅在 Windows 下有效。
    *   **对策**：在代码开头增加平台检测 `if sys.platform == 'win32':`，如果是 Linux/Mac 则使用 `tty` 模块或暂时禁用此功能并给出提示。
*   **风险**：后台线程未及时退出导致资源泄漏。
    *   **对策**：将线程设置为 `daemon=True`，确保主程序结束时它会自动销毁。
