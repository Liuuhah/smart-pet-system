# AI 管家双模态交互与人格化设计指南

**版本**: Phase 1.4  
**状态**: ✅ 已完成  
**完成日期**: 2026-04-25  
**优先级**: 🔴 P0 (高优先级 - 提升用户体验)  
**目标**: 解决 AI 回复“上纲上线”的问题，实现“闲聊模式”与“专业问诊模式”的智能切换，打造养成系、有温度的宠物管家。

---

## 1. 问题背景 (User Feedback)

### 1.1 用户痛点
*   **过度反应**：用户仅输入“你好”，AI 却输出长篇大论的医疗建议和紧急程度分级，导致交互压力极大。
*   **缺乏人情味**：AI 表现得像一个冷冰冰的诊断机器，无法进行日常的情感交流或闲聊。
*   **角色固化**：无论用户说什么，AI 始终处于“严肃兽医”状态，缺乏灵活性。

### 1.2 核心需求
用户希望 AI 具备**“双重人格”**：
*   **闲聊时**：随和、亲切、像老朋友一样自然。
*   **问诊时**：专业、严谨、提供结构化的医疗参考。

---

## 2. 实施方案：意图识别与动态 Prompt

### 2.1 核心思路
在 `PetHealthAdvisor` 中增加一个轻量级的**意图识别层**。根据用户输入的关键词，动态调整发送给 LLM 的 System Prompt 指令和 Temperature（温度值）。

### 2.2 模式定义

| 模式 | 触发条件 | System Prompt 侧重 | Temperature | 预期表现 |
| :--- | :--- | :--- | :--- | :--- |
| **闲聊模式** | 无医疗关键词（如：你好、在吗、今天天气不错） | 强调“贴心管家”、“情感陪伴”、“轻松语气” | 0.8 | 简短回应，带点幽默感，不主动提建议 |
| **专业模式** | 包含医疗关键词（如：吐、拉、不吃、发烧、药） | 强调“兽医专家”、“5W诊断”、“结构化输出” | 0.4 | 详细分析，给出护理建议和就医指引 |

### 3.1 `src/ai_assistant/pet_health_advisor.py` 修改方案

#### 任务 A：新增 `_detect_intent()` 方法
用于判断当前对话属于哪种模式。

```python
def _detect_intent(self, user_input: str) -> str:
    """
    简单的关键词意图识别。
    返回 'chat' (闲聊) 或 'consult' (问诊)。
    """
    medical_keywords = ["病", "痛", "吐", "拉", "吃", "喝", "药", "医", "诊", "症状", "发烧", "咳嗽", "瘦", "胖"]
    # 如果包含任何医疗关键词，则进入专业模式
    if any(kw in user_input for kw in medical_keywords):
        return "consult"
    return "chat"
```

```python
def consult(self, user_input: str) -> str:
    intent = self._detect_intent(user_input)
    
    if intent == "chat":
        # 闲聊模式：注入轻松的指令
        context_instruction = "\n\n【当前状态：闲聊】\n用户只是在打招呼或闲聊。请保持轻松、随和的语气，像朋友一样简短回应，不要输出任何医疗建议格式。"
        temperature = 0.8
    else:
        # 专业模式：注入严谨的指令
        context_instruction = "\n\n【当前状态：专业问诊】\n用户提到了健康问题。请严格按照兽医标准，提供结构化诊断、护理建议和紧急程度评估。"
        temperature = 0.4
    
    # 调用底层接口，传入动态指令和温度
    return self.compressor.chat(user_input, system_suffix=context_instruction, temperature=temperature)
```

## 4. 验收标准 (Acceptance Criteria)

1.  **闲聊测试**：
    *   输入：“你好呀”。
    *   **预期**：AI 回复：“你好呀！豆豆今天看起来精神不错呢~ 有什么想聊的吗？”（简短、亲切、无医疗格式）。
2.  **问诊测试**：
    *   输入：“豆豆最近有点拉肚子”。
    *   **预期**：AI 回复包含“🟡 紧急程度”、“🏠 家庭护理”等结构化内容，且语气专业。
3.  **混合测试**：
    *   先聊“今天天气真好”，再聊“它好像不爱吃饭了”。
    *   **预期**：AI 能平滑地从闲聊切换到专业诊断，且不显得突兀。

## 5. 进阶思考：养成系元素 (Future Work)

为了让 AI 更有“养成感”，未来可以考虑：
*   **记忆关联**：如果用户昨天聊了“豆豆去公园了”，今天打招呼时 AI 可以主动问：“豆豆昨天玩得开心吗？”
*   **情绪感知**：如果用户语气焦急（使用感叹号或“急”字），即使没有医疗关键词，也自动提高响应优先级。

---

## 6. 风险提示
*   **误判风险**：简单的关键词匹配可能会漏掉一些隐晦的病情描述（如“它看起来没精神”）。
*   **对策**：初期可以使用关键词，如果发现误判率高，可以改为让 LLM 在内部先进行一次“意图分类”（即先发一个极短的请求问 LLM 这是什么意图，再决定怎么回）。

1. **低成本高效益**：
   - 基于关键词的意图识别，实现成本低但效果显著

   - 通过 `system_suffix` 实时调整 AI 的语气和行为准则

   - 闲聊模式用高温度（0.8），增加灵动性和创造性
   - 专业模式用低温度（0.4），提高严谨度和确定性
   - 从底层控制 AI 的“严谨度”与“灵动性”

4. **用户体验提升**：
   - 闲聊时：简短、亲切、无医疗格式
   - 问诊时：结构化、专业、详细分析
   - 平滑切换，不显得突兀

### 7.6 使用说明

#### 自动切换流程
用户无需任何额外操作，系统会自动完成模式切换：

1. 用户输入文本
2. 系统自动识别意图（闲聊 / 问诊）
3. 根据意图选择合适的 temperature 和 System Prompt
4. 调用 LLM 生成回复
5. 恢复原始 System Prompt，准备下一次对话

#### 开发者注意事项
- **不要手动判断意图**：所有对话都应调用 `consult()` 方法，由系统自动识别
- **temperature 范围**：建议在 0.2-0.9 之间，过低会导致回复呆板，过高会导致回复不稳定
- **关键词扩展**：如果发现某些医疗描述未被识别，可以在 `_detect_intent()` 的 `medical_keywords` 列表中添加新词

### 7.7 问题解决

#### 修复的问题
- **过度反应问题**：用户仅输入“你好”，AI 不再输出长篇大论的医疗建议
- **缺乏人情味问题**：闲聊模式下 AI 表现得更亲切、随和，像老朋友一样
- **角色固化问题**：AI 能根据用户输入智能切换“闲聊”和“专业”两种状态

#### 潜在风险与对策
- **风险**：简单的关键词匹配可能会漏掉一些隐晦的病情描述（如“它看起来没精神”）。
  - **对策**：已在关键词列表中包含“精神”、“异常”等词汇。如果未来发现误判率高，可以升级为 LLM 意图分类。
  
- **风险**：频繁修改 System Prompt 可能影响性能。
  - **对策**：每次调用后立即恢复原始 System Prompt，确保状态一致性。实际测试显示性能影响可忽略不计。
